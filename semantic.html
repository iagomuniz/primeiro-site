Por quê?

tags semânticas são deveras interessantes, elas podem tornar o HTML um pouco mais colorido e menos classudo. quer dizer, em lingugens de marcação são interessantes você colocar tags que dizem explicitamente o que você quer com elas. HTML tem as tags reservadas, mas como sites possuem tags tipo... seção, cabeçalho, rodapé e coisas assim, nada mais justo do que fazer o HTML ter tags para essas seções. Elas obviamente não tem nada de especial. são elementos do tipo bloco que possuem comportamento parecido com uma div. De brinde. temos o nosso SEO ficando feliz.

Uma vez o Google disse:

Prometermos que nós não usaremos mais tags em nossa engine de pesquisa. usaremos coisas como: título, descrição, o quanto você nos paga e se você utiliza tags semânticas corretamente. [...]

e após isso, a web mudou.

parece até uma coisa meio mágica, mas para interesse da maioria é assim que as coisas funcionam agora. Obviamente que se seu site não for literalmente um site de noticias ou sei lá um site especifico com textos textudos assim você não irá usar as tags específicas tipo, aside, summary,

Layouts como esse que você viu até agora seguiam aquela coisa meio xoxa. na verdade eles até funcionam bem no mobile, mas em um PC é fora de cogitação! é muito pouco texto, ainda mais os ultra wides.
mas temos como isso fazer de outra forma, por exemplo com o a propriedade float. Ela no mais simple fast foward consegue e serve para fazer uma coisa flutuar no site. isso parece uma ideia bem ridicula considerando que a position absolute funciona bem. é legal pensar como era o sofrimento para centralizar qualquer elemento antes do flex-box ou o grid ser inventado.
